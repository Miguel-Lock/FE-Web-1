Project Experience:

  • ScriptOS application for the USCCB

    • For my Senior Capstone project (2024-2025), I am working in a team of six developers to create a web application for the United States Conference of Catholic Bishops. For background, a current stamp of approvaal - a declaration of conformity - exists. Current methods for publishers to obtain this approval are slow and inefficient. Long, unweildy email threads facilitate communiction between the publishers and USCCB Consultants. If any changes need to be made, the publishing company has to ensure the proper edits are made, re-format the book, and send it back to the USCCB. Few publishing companies are able to undergo the financial burden of the process, which currently takes years.

    Our solution is a custom-built web application that facilitates communication between the publishers and USCCB. Publishers can submit a proposal, which can be approved and become a project. Instead of submitting a completed book, a publisher can submit a single chapter at a time. The publishers and consultants talk through a comments thread built into the PDF. If any changes need to be made, the publisher can submit a new version of each chapter. Once all chapters are approved, the project is presented to the bishop for final approval.

    My role in the SCRUM team is assistant SCRUM master. My role in the project is the cloud manager and assistant front-end developer. We use React for the front-end, Ruby on Rails for the back-end, and MSSQL for the database. Some of the work I am most proud of is replacing existant redundant code to implement tempates for code reusability. Additionally, my work has been key in developing the wireframes and implementing my work on the wireframes in the project.
